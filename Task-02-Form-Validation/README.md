## 🎯 Task Objective

Implement both **client-side JavaScript validation** (inline, real-time feedback) and **server-side validation** (Express route) for the registration form. Use a temporary in-memory `Map` store before MongoDB is introduced.

---

## 🧠 Concepts Covered

| Concept | Description |
|---------|-------------|
| **Client-Side Validation** | Inline JS — real-time field validation feedback |
| **Server-Side Validation** | Express route validates input before saving |
| **Temp Storage** | In-memory `Map` as a mock database |
| **Error Rendering** | Pass validation errors back to EJS view |
| **UX Feedback** | Show success/error messages dynamically |
